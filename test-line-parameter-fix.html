Line Parameter Fix Test

Problem Identified

❌ Before Fix

Issue: Frontend was converting default values (1, 1.0) to null

// Frontend logic (WRONG) const numValue = parseFloat(input.value); values[param.id] = (input.value.trim() === '' || numValue === 0) ? null : numValue; // This converted parallel=1 and df=1.0 to null!

Result: Backend received null values even when frontend showed 1 and 1.0

✅ After Fix

Solution: Only convert empty strings to null, not default values

// Frontend logic (FIXED) const numValue = parseFloat(input.value); values[param.id] = (input.value.trim() === '') ? null : numValue; // Now parallel=1 and df=1.0 are preserved!

Result: Backend receives actual values (1, 1.0) from frontend

Backend Processing Fix

Backend Changes:

  • ✅ Always include parallel and df parameters in pandapower call
  • ✅ Use default values (1, 1.0) when frontend sends null/empty
  • ✅ Use actual values when frontend sends valid numbers
  • ✅ Proper error handling for invalid conversions
# Backend Processing (FIXED) for param in optional_params: value = in_data[x].get(param) if value is not None and value != 'None' and value != '': try: if param == 'parallel': line_params[param] = int(value) # Use actual value else: # df line_params[param] = float(value) # Use actual value except (ValueError, TypeError): # Fallback to defaults if conversion fails line_params[param] = 1 if param == 'parallel' else 1.0 else: # Use defaults when parameter is None/empty line_params[param] = 1 if param == 'parallel' else 1.0
