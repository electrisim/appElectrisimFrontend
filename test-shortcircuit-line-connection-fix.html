Issue Fixed

Original Frontend Error:

Error processing line. Please check the console for more details: shortCircuit.js:1059 Line mxCell#188 is not properly connected (anonymous) @ shortCircuit.js:1059 (anonymous) @ shortCircuit.js:539 (anonymous) @ ShortCircuitDialog.js:98 await in (anonymous) applyButton.onclick @ Dialog.js:192

Root Cause Analysis:

  • Inconsistent Connection Detection: `shortCircuit.js` used edge-based connection detection while `loadFlow.js` used direct source/target approach
  • Over-strict Validation: The `getConnectedBuses` function threw errors for lines that didn't have exactly 2 edges
  • Different mxGraph Object Structure: Lines in mxGraph have `source` and `target` properties directly, not necessarily through `edges` array
  • Processing Difference: Load flow worked because it used the correct connection detection method

Solution Applied:

‚úÖ Fixed `getConnectedBuses` function to use direct source/target approach like in `loadFlow.js`

‚úÖ Updated `validateBusConnections` to work with the new connection detection method

‚úÖ Removed over-strict edge count validation that was causing false positives

‚úÖ Made connection detection consistent between loadFlow and shortCircuit

Code Changes Made

Before (Edge-based approach causing errors):

const getConnectedBuses = (cell) => { if (!cell.edges || cell.edges.length < 2) { throw new Error(`Line ${cell.mxObjectId} is not properly connected`); } const [edge1, edge2] = cell.edges; const getBusId = (edge) => { if (edge.target && edge.target.mxObjectId !== cell.mxObjectId) { return edge.target.mxObjectId.replace('#', '_'); } else if (edge.source && edge.source.mxObjectId !== cell.mxObjectId) { return edge.source.mxObjectId.replace('#', '_'); } throw new Error(`Invalid edge connection for line ${cell.mxObjectId}`); }; return { busFrom: getBusId(edge1), busTo: getBusId(edge2) }; };

After (Direct source/target approach matching loadFlow.js):

const getConnectedBuses = (cell) => { // For lines, use the direct source/target approach like in getConnectedBusId(cell, true) // This matches the approach used in loadFlow.js return { busFrom: cell.source?.mxObjectId?.replace('#', '_'), busTo: cell.target?.mxObjectId?.replace('#', '_') }; };

Updated Validation Function:

const validateBusConnections = (cell) => { const connections = getConnectedBuses(cell); if (!connections.busFrom || !connections.busTo) { throw new Error(`Line ${cell.mxObjectId} is not connected to two buses`); } return true; };
